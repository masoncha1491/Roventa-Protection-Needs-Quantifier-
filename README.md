# 保障需求量化系统 | Protection Needs Quantifier

现在已经整理为单文件版本。

## 需要上传的文件

只需要上传根目录中的 `index.html`。

不需要上传：

- `protection-needs-quantifier.html`
- `protection-needs-quantifier-zh.html`
- `protection-needs-quantifier-en.html`
- `insurance-gap-tool.html`
- `.uploads`

## 上传到 GitHub

1. 在 GitHub 新建一个仓库
2. 只把 `index.html` 上传到仓库根目录
3. 提交并推送到 `main` 分支

## 开启 GitHub Pages

1. 打开仓库
2. 进入 `Settings`
3. 打开 `Pages`
4. 在 `Build and deployment` 中选择：
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`
   - `Folder`: `/ (root)`
5. 保存后等待几分钟

## 访问方式

开启 GitHub Pages 后，通常会得到类似下面的网址：

`https://你的GitHub用户名.github.io/仓库名/`

打开后会直接进入单文件版本页面，并可在页面右上角切换 `中文` / `English`。

## 说明

- `index.html` 已包含中文和英文界面
- `index.html` 已包含计算逻辑与 PDF 下载功能
- 如果后续要绑定自定义域名，也可以继续在 GitHub Pages 里设置
